# מדריך מפורט — צד הלקוח (React) של פרויקט Beton

מסמך זה מסביר בעברית, בצורה מסודרת, **כל חלק** בצד הלקוח: המבנה, כל דף ולמה
הוא מיועד, הרכיבים, ובחירות העיצוב והצבעים. (ההסבר על שכבת השרת והתשובות
לשאלות נמצאים בקובץ `ANSWERS_HE.txt`.)

---

## 1. ארכיטקטורה כללית

האפליקציה בנויה כ-**SPA** (Single Page Application): דף HTML אחד (`index.html`)
שלתוכו React מצייר את כל המסכים, ומחליף ביניהם בלי לרענן את הדף (בעזרת
`react-router-dom`). התקשורת עם השרת היא דרך **REST API** (קריאות `fetch`).

זרימת הטעינה:
```
index.html  →  src/main.jsx  →  <BrowserRouter> + <AuthProvider> + <App/>
                                          │
                                          └─ App.jsx מגדיר את הניתוב לכל דף
```

---

## 2. מבנה התיקיות (src/)

| נתיב | תפקיד |
|------|-------|
| `main.jsx` | נקודת הכניסה. עוטף את האפליקציה ב-Router וב-AuthProvider |
| `App.jsx` | מפת הניתוב — איזה URL מציג איזה דף, ומי מוגן בהתחברות |
| `index.css` | מערכת העיצוב המלאה — צבעים, מרווחים, רכיבים (מתועד שורה-שורה) |
| `config/constants.js` | קבועים: תפקידים (לקוח/קבלן), רשימת אזורים, מפתח אחסון |
| `api/apiClient.js` | עטיפה אחידה לכל קריאת HTTP + טיפול מרוכז בשגיאות |
| `api/betonApi.js` | פונקציות API לפי ישות (לקוחות, קבלנים, בקשות, הצעות, lookup) |
| `context/AuthContext.jsx` | ניהול "מי מחובר": `register` / `login` / `logout` |
| `components/` | רכיבים לשימוש חוזר (ראה למטה) |
| `pages/` | חמשת הדפים (ראה למטה) |

### הרכיבים (`components/`)
- **Navbar** — סרגל עליון קבוע עם ניווט ופרטי המשתמש המחובר.
- **Footer** — תחתית עם זכויות וקישור לתיעוד ה-API.
- **Alert** — הודעה צבעונית (הצלחה / שגיאה / מידע).
- **FormField** — שדה טופס אחיד (תווית + קלט + הסבר + שגיאה). תומך input/select/textarea.
- **ProtectedRoute** — שומר על דפים שדורשים התחברות ותפקיד מתאים.
- **LocationPicker** — איסוף מיקום: כתובת + קו רוחב/אורך + כפתור "אתר מיקומי".

## 3. הדפים — מה כל אחד עושה ולמי הוא מיועד

### דף הבית — `HomePage` (`/`)
**מיועד ל:** כל מבקר. **מטרה:** להסביר בקצרה את הרעיון ולהפנות לשני המסלולים
(לקוח / קבלן). אם המשתמש כבר מחובר, הכרטיסים מובילים ישירות לפעולה לפי תפקידו.

### דף הרשמה — `RegisterPage` (`/register`) ← סעיף א'
**מיועד ל:** משתמש חדש. **מטרה:** לפתוח חשבון כ"לקוח" או "קבלן". הטופס שולח
שם + טלפון לשרת (`POST /customers/` או `POST /contractors/`), ואז מחבר את
המשתמש אוטומטית ומעביר אותו לדף הפעולה שלו.

### דף התחברות — `LoginPage` (`/login`) ← סעיף ב'
**מיועד ל:** משתמש קיים. **מטרה:** כניסה חוזרת. בוחרים תפקיד ומזינים את הטלפון
שאיתו נרשמו; ה-AuthContext מאתר את הרשומה בשרת.

> **למה אין סיסמה?** בשרת הנוכחי הטבלאות `Customers`/`Contractors` מכילות רק
> שם וטלפון. לכן "התחברות" = איתור לפי טלפון. כשתתווסף אבטחת סיסמאות בשרת,
> צריך לעדכן **רק** את `AuthContext.jsx`.

### דף בקשת לקוח — `CustomerRequestPage` (`/customer/request`) ← סעיף ג'
**מיועד ל:** לקוח מחובר בלבד (מוגן ב-`ProtectedRoute`). **מטרה:** לפרסם בקשה
לבטון. שדות: מטרה (נטענת מהשרת), כמות, אזור, ומיקום (כתובת + קואורדינטות עם
כפתור "אתר מיקומי"). נשלח ל-`POST /concrete-requests/`.

### דף הצעת קבלן — `ContractorRequestPage` (`/contractor/request`) ← סעיף ד'
**מיועד ל:** קבלן מחובר בלבד. **מטרה:** לפרסם עודפי בטון. שדות: סוג בטון (עם
תווית קריאה שנבנית מטבלאות העזר — חוזק/סומך/אבן/מטרה), כמות, **זמן תפוגה**
(קריטי בבטון!), אזור, ומיקום. נשלח ל-`POST /contractor-offers/`.

## 4. בחירות העיצוב והצבעים (ולמה)

ערכת הצבעים מתועדת במלואה בראש `src/index.css`. בקצרה:

| צבע | ערך | תפקיד ולמה נבחר |
|-----|-----|-----------------|
| אפור-פחם כהה | `#1f2933` | צבע ראשי = **הבטון**. יציבות, מקצועיות, רצינות |
| ענבר/כתום | `#f59e0b` | הדגשה = צבע **הבטיחות באתרי בנייה** + תחושת דחיפות (בטון מתקשה) |
| אפור בהיר | `#f4f5f7` | רקע = ניקיון ואווריריות, מראה מודרני |
| ירוק | `#10b981` | הצלחה (אוניברסלי) |
| אדום | `#ef4444` | שגיאה (אוניברסלי) |

**הרציונל:** צבע ניטרלי (אפור = בטון) שולט ברוב השטח, וההדגשה הכתומה מופיעה
**רק** בנקודות הפעולה החשובות — כך העין מובלת בדיוק לאן שצריך, והאתר נשאר
נקי, רגוע ואלגנטי. הכתום גם "מספר את הסיפור" של דחיפות הזמן, שזה לב המוצר.

**עקרונות נוספים:**
- **טיפוגרפיה:** הפונט "Heebo" — עברי, מודרני, נקי וקריא, מתאים ל-RTL.
- **RTL:** כל האתר מימין-לשמאל, מותאם לעברית.
- **רכיבים "כרטיסיים":** פינות מעוגלות (12–20px) וצללים רכים → תחושת עומק עדינה.
- **רספונסיביות:** מסתדר יפה גם במובייל (שדות נערמים, כותרות קטֵנות).
- **חוויית משתמש:** הודעות הצלחה/שגיאה ברורות, ספינר טעינה, ולידציה עם הסברים,
  וכפתור "אתר מיקומי" שחוסך הקלדת קואורדינטות.

## 5. איך מריצים

ראה הוראות מלאות ב-`README.md`. בקצרה: `npm install` → להעתיק `.env.example`
ל-`.env` → `npm run dev` → לפתוח http://localhost:5173 (כששרת ה-FastAPI רץ במקביל).
